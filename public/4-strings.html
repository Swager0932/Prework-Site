Lesson 4: Strings

Say Something Good

String manipulation is one of the cornerstones of programming. Which makes sense since so much of what we do on the internet is just some kind of text.

Dynamic Template Strings

It's common to only want a piece of a string to change. Instead we use Template Strings, which allow us to interpolate (meaning "insert") data directly. Template Strings can also be called Template Literals.

Creating template strings

When creating a template string instead of using " or ', you surround your text with backticks ``. You get those by hitting shift and the key above your tab key with your left pinky. 

let greeting = `Hello!`

That's technically a template string, it's just boring, the entire thing is non-dynamic code. To spice it up, let's add a variable. Perhaps a name? In order to insert a chunk of code into the template, you have to wrap it inside a ${ } operator. You'll know you did it right if the code changes color. 

let name = "Jay"
let greeting = `Hello ${name}!`

You can interpolate multiple pieces into a single string, and I meant it when I said "chunks of code." You aren't limited to only variables. Try playing around with some templates to get the hang of it. Be careful about your spaces! 

let name = "Jay"
let emphasis = "!"
let greet = `Hello, ${name}${emphasis}`

console.log(greet)

let price = `$${4 + 2} dollars`
console.log(price)

Since a template is enclosed by backticks, you can use regular quotes inside without issue. 

let quotes = `"All quotes" 'work!'`
console.log(quotes)

Concatenation

You can use something called concatenation to literally add strings together, but it's a little clunkier in most cases. Templates are more common now.

The length Property

Sometimes, (like in the next lesson) you'll want to know how long a string is. Luckily, every string has a built in property length. All you need to do is add .length to the end of any string. Just make sure it's after the quote! 

console.log("hi".length) // 2
console.log('hello'.length) // 5
console.log(`hey`.length) // 3

let str = "hey hey"
console.log(str.length) // 7

let name = 'bo'
console.log(`hi ${name}`.length) // 5

Look at that template. The length of the string is the final string, not the pre-computed version with the code. Also remember that length counts the number of characters, so spaces and punctuation add up.

String Indexes

We call text strings because it's like you're stringing a bunch of individual characters together. And JavaScript has an order number for each one of them. We call that number the index.

Each index is a number, and only refers to a single character. Instead of starting at 1, indexes start at 0. The "0th" index is always the first character of the string.

You can access the value at an index by putting some square brackets and the index number at the end of the string. 

let str = "hello"
console.log(str[0]) // h
console.log(str[2]) // l
console.log(str[4]) // o

// the bracket can use any expression
// as long as it evaluates to a number
console.log(str[2]) // l
console.log(str[1+1]) // l

// you can access on a plain string too
console.log("wow"[0]) // w

Out of bounds

Be careful with indexes. If you overshoot and list an index that isn't in the string, you'll get undefined instead of a character. 

let str = "hello"
console.log(str[100]) // undefined

Getting the last character

Here's a fun trick. Since we know that the last index will always be 1 less than the length (because length starts counting from 1, and indexes start from 0), there's a clever bit we can do to get the last character of the string. 

const getLastChar = (str) => {
  return str[str.length - 1]
}

console.log(getLastChar("Hey!")) // !
console.log(getLastChar("Oh?")) // ?

Here's a fun challenge, how would you always find the second to last character in a string?

Immutable Strings

I don't want to get too in depth here, but strings themselves are immutable, meaning they can't be mutated or changed in any way. You can reassign a variable from one string to another, but you can't alter an existing string. That means you can only use the indexes to read values. 

let same = "hello"
same[0] = "Z"

console.log(same)
// still says hello, not Zello

// but something like this is fine
let str = "hi"
str = `${str[0]}${str}${str[1]}`
console.log(str) // hhii

Common String Methods

Strings have built in properties like length, but they also have built in functions too! You access them with dot notation just like you do with length. Just don't forget the parentheses!

.toUpperCase() and .toLowerCase()

These are for returning copies of entire strings as upper or lower case. 

let greet = "hello"
let shoutedGreet = greet.toUpperCase()
console.log(shoutedGreet)
// HELLO

let loud = "AHHHH"
let sigh = loud.toLowerCase()
console.log(sigh)
// ahhhh

// remember, the originals are not changed
console.log(greet) // hello
console.log(loud) // AHHHH

Some common use cases are, only capitalizing the first letter, and lowercasing 2 strings in order to check equality. Because remember, === is case sensitive, so "Hello" !== "hello"

.slice()

Sometimes you only want a part of a string. .slice() returns a copy of a string. You can specify both starting index, which is inclusive, and optionally an ending index, which is exclusive (up to, not included). If no end index is given, slice will copy to the end of the string. If no indexes are given, everything is copied. 

let str = "Here I Am"
let copy = str.slice()
let here = str.slice(0,4)
let i = str.slice(5,6)
let am = str.slice(7)

console.log(copy) // Here I Am
console.log(here) // Here
console.log(i) // I
console.log(am) // Am

.includes()

This method is returns a boolean, true if the argument string is in the owner string, and false otherwise. 

let str = "Look at me!"
console.log(str.includes("Look")) // true
console.log(str.includes("at me")) // true
console.log(str.includes("ook ")) // true
console.log(str.includes("us")) // false

When searching strings, it's a good idea to lowercase everything. 

const doesContain = (mainStr, subStr) => {
  return mainStr
    .toLowerCase()
    .includes(subStr.toLowerCase())
}

let wonky = "DoEs ThIs WoRk?"
let work = "work"
let isTrue = doesContain(wonky, work)
console.log(isTrue) // true

Chaining

Now you may be wondering what this nonsense is: 

return mainStr
  .toLowerCase()
  .includes(subStr.toLowerCase())

It's called chaining, it's like a combo from a video game. All these methods can be called on strings, right? So if a method returns a string, we can immediately call another method on it. Here's the same logic, but spread out. You can use multiple lines as long as you start each line with the dot. 

// With chaining
const doesContain = (mainStr, subStr) => {
  return mainStr
    .toLowerCase()
    .includes(subStr.toLowerCase())
}

// no chaining
const doesContain = (mainStr, subStr) => {
  let lowMainStr = mainStr.toLowerCase()
  let lowSubStr = subStr.toLowerCase()

  return lowMainStr.includes(lowSubStr)
}
