Lesson 5: Loops

Do It Again And Again

Sometimes we need our programs to do something over and over again. Like a game with multiple rounds, or filter through every student in a class. If you ever want a chunk of code to be run multiple times, you'll need some kind of a loop. There are lots of looping options to choose from, but today we're going to focus on one of the classics: the for loop.

Structure of a for loop

A for loop has 2 main sections: the head and the body. The body is simple, it's a chunk of code that we want to run with each iteration of the loop (a single step of a loop is called an iteration). The head is the more interesting part. It's made up of 3 sections, The initialization, the Condtion, and finally the Incrementor. The first two always have semi colons (;) after them, but the third does not.

How for Loops Work

What's happening is that our loop first needs an iterator value to work. Due to tradition (and because programmers can't be bothered to type "iterator"), the iteration value is pretty much always just i.

The loop then uses that i to decide if it should run the loop's body or not by checking the condtion. If the condtion is true, the loop body runs once.

But, you may be wondering, how do we ever change the value of i so that our loop will stop? That's the job of the increment section. After each iteration is run, this chunk is exectuted and adds 1 to the i value. That's what that ++ operator does. It just adds 1 to whatever value it's next to. It would be the same as writing i += 1.

Then the cycle repeats! The condition is checked again, if it passes the body is executed, and then the iterator goes. The loop will finally stop once the condition is no longer true. Below is an image of this entire process (except it only goes to 2, not 5, so we don't repeat too much text).

Versatile Loops

Now that you undertsand the basic structure, know that it's extremely flexible! We can start on any number, end on any number, and even go up or down by any number!

Loop through a string

If you were wondering why I started on 0, this is why! A lot of things in JavaScript are 0-indexed, for example, strings. So, by using i and bracket notation, and the string's length property, we can iterate through a string. 

let str = "Howdy"
for (let i = 0; i < str.length; i++) {
  console.log(str[i])
}

And that logic will work on any string because every string index starts at 0, and the loop will always stop 1 before the length property. Why 1 before? Because we're using <, not <=. The reason why that's so important is becuase if you try to reference an index of a string that doesn't exist with brackets, you'll just get undefined.

Up and Down Faster!

While almost 99% of the time our loops will just go up by 1, there's no rule. You can go up by any value you like. Lets start at 1, and go up by 2, so we can log every odd number from 1 to 10. 

for (let i = 1; i < 10; i += 2) {
  console.log(i)
}

By that same token, there's no need to only go up either. The increment section can also be used to decrement as well. Since i++ goes up by one, i-- goes down by one. And no, it's just those two, there's no other shorthand. For anything else just use i -=. But let's do a count down! Look at how we alter our start and stop points! 

for (let i = 3; i > 0; i--) {
  console.log(i)
}
console.log('GO!')

Break, Continue

And last but not least, for loops have the ability to bail. If you use the continue keyword, the iteration will skip whatever was left inside, and the loop will move to the next iteration. Look how we use continue to log everything but 3. 

for (let i = 1; i < 6; i++) {
  console.log('hi')
  if (i === 3) {
    continue
  }
  console.log(i)
}
console.log('Done')

But sometimes that's not enough. If you want to completely abandon not only the spot in the current iteration, but all iterations after that too, use break instead. Lets look like we'll loop up to 10, but bail out at after 5 with the break keyword. 

for (let i = 1; i < 10; i++) {
  if (i === 6) {
    break
  }
  console.log(i)
}

Return

You can of course use loops in a function, and if you use return in a loop, it behaves a lot like break. Except instead of just bailing on the loop, anything outside the loop in the function gets skipped. Below, note that neither the looped 5 nor the "I should not appear" message get logged. We only see the 5 because we are logging the variable it got stored in. 

const loopAndGive5 = () => {
  for (let i = 1; i < 10; i++) {
    if (i === 5) {
      return i
    }
    console.log(i)
  }
  console.log("I should not appear")
}

let five = loopAndGive5()
console.log('OUTSIDE!')
console.log('five here:', five)
